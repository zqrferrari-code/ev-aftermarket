路由结构

新增路由（与现有模式一致）
现有
/:market/problems → 所有车型问题汇总
/:market/problems/[model] → 车型问题详情
新增
/:market/warnings/[brand] → 品牌警告灯汇总（BYD 58条）
/:market/warnings/[brand]/[model] → 车型警告灯详情

数据库表设计

mf_nv_warning_lights
id serial PK
brand_id varchar(50) → brands.brand_id
model_id varchar(100) nullable → models.model_id
category varchar(50) // 电池与充电 | 制动系统 | 驾驶辅助…
name_en varchar(200) // "Battery Fault Warning"
name_cn varchar(200) // "动力电池故障警告灯"
severity varchar(10) // INFO | WARNING | CRITICAL
description_en text // 含义说明
causes json // string[]
can_drive varchar(20) // yes | no | caution
action_en text // 建议操作
is_model_specific boolean // 车型专属 vs 品牌通用
mf_nv_warning_light_dtc_links（关联表）
warning_light_id int → warning_lights.id
dtc_id int → dtcs.dtc_id
PRIMARY KEY (warning_light_id, dtc_id)
