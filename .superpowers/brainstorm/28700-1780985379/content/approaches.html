警告灯功能 — 3 个实现方案

选择你倾向的整体架构方向

A

警告灯独立模块

新建独立路由 /[market]/warnings/[brand] 和 /[market]/warnings/[brand]/[model]，与 DTC、charging、service 并列，作为完全独立的功能模块。

优点

  • 结构清晰，易于 SEO 独立优化
  • 不影响现有页面
  • 可独立扩展到其他品牌

缺点

  • 用户需要多一层导航才能发现
  • 与 DTC 的关联靠链接维持，不够紧密
B

嵌入 Model 页面

在现有 /[market]/models/[slug] 页面里增加"Warning Lights"section，不新建路由，警告灯作为 model 详情的一部分展示。

优点

  • 用户在 model 页就能看到所有信息
  • 无需额外路由

缺点

  • model 页会更长更重
  • 无法做品牌级汇总页
  • SEO 价值分散
C

独立模块 + Model 页摘要（推荐）

新建独立的 warnings 路由（品牌 + 车型两层），同时在 model 页的 "More Resources" 区域加入警告灯入口链接。结构与现有 problems/charging/service 完全一致。

优点

  • 与现有架构模式完全一致
  • 独立 SEO 价值（"BYD Atto 3 warning lights"）
  • 品牌汇总页可做内链枢纽
  • model 页保持轻量，通过链接引流

缺点

  • 新建文件较多（brand page + model page）
