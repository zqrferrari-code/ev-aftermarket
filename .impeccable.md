## Design Context

### Users
EV car owners (primary) and auto technicians / used car dealers (secondary). Owners are the dominant audience — they arrive from Google after seeing a dashboard warning light, often outdoors on mobile, sometimes anxious (especially for Critical codes). Technicians want fast data lookup.

### Brand Personality
Practical, reliable, authoritative. Three words: **technical, clear, trustworthy**. The interface should feel like a well-maintained workshop manual — not cold or clinical, but precise and confidence-giving.

### Aesthetic Direction
- **Theme:** Light. Users are often in bright environments (parking lots, roadside). Warm off-white base (`oklch(95.5% 0.006 60)`) not pure white.
- **Typography:** Barlow Condensed (headings, labels, DTC code titles) + Barlow (body) + JetBrains Mono (DTC code values). Technical but readable, not sterile.
- **Color:** Warm-tinted neutrals. Green (`oklch ~50% 0.18 145`) as brand accent (EV / eco). Severity colors: red for Critical, amber for Warning, blue for Info — muted, not garish.
- **Anti-references:** No glassmorphism, no dark mode with glowing accents, no generic SaaS card grids, no rounded-corner icon-above-heading layouts.

### Design Principles
1. **Severity is visual, not just a badge** — Critical pages feel different from Info pages. The safety banner, amber drive-box, and color palette shift with severity.
2. **Most important answer in 3 seconds** — Code + severity + one-line description visible immediately on load. "Can I still drive?" answered before the fold on mobile.
3. **Mobile-first, content-first** — Padding tightens on small screens, layout adapts (list rows go 2-line), nothing is hidden or amputated.
4. **Earned density** — Lists are compact but not cramped. Cause numbers, action steps, and case metadata use typographic hierarchy rather than heavy borders or cards-within-cards.
5. **No decorative chrome** — Every visual element (badge, banner, numbered pill) serves the content. No side-stripe accents, no gradient text, no sparklines.

### Key Components (approved mockup)
- **Site header:** `EVAftermarket` logo (green accent on "Aftermarket") + market badge (AU/UK/etc)
- **List page:** Hero with h1 + description + stat row (total / critical / warning counts) → filter chips → compact 2-line rows (code + badge / description)
- **Detail page:** Breadcrumb → optional safety banner (Critical only) → hero (big mono code + badge + h1 + system label) → drive-box (amber, Warning/Critical) → Likely Causes (numbered green pills) → What To Do (numbered circle steps with title+body) → Real-World Cases (source meta + title + body + resolution) → Related codes (chip grid) → Disclaimer
- **Fonts:** Load from Google Fonts — `Barlow:wght@400;500;600;700`, `Barlow+Condensed:wght@600;700`, `JetBrains+Mono:wght@500;700`
